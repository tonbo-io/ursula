title: Install
---

This page gets a **single Ursula node** running on your machine so you can try the API. There are two ways to do it:

| Pick | When |
| ---- | ---- |
| **Docker** | You just want to try Ursula. One command, no toolchain. |
| **Build from source** | You want local binaries, plan to hack on Ursula, or can't run Docker. |

Both start the same in-memory, non-replicated node on port `4437` — perfect for a first look, with nothing persisted across restarts. For a production cluster, skip ahead to [Deploy a Cluster](/docs/deploy-cluster).

## Docker

Every release publishes a multi-arch image to GHCR:

```bash
docker run --rm -p 4437:4437 ghcr.io/tonbo-io/ursula:0.3.1
```

The image runs as a non-root user and contains both `ursula` (the server) and `ursulactl` (the operator CLI). `latest` tracks the main branch; pin a version tag for anything you keep running.

## Build from source

You need rustup (the repo pins the right nightly via `rust-toolchain.toml`), a C compiler, and `pkg-config`.

On macOS:

```bash
brew install rustup pkg-config
rustup-init -y
```

On Debian or Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y build-essential pkg-config curl
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Then clone, build, and run:

```bash
git clone https://github.com/tonbo-io/ursula.git
cd ursula
cargo run --release --bin ursula
```

To install the binaries somewhere instead of running in place:

```bash
cargo build --release -p ursula -p ursula-ctl
```

They land at `target/release/ursula` and `target/release/ursulactl`.

## Verify

With the node running, from another terminal:

```bash
curl http://127.0.0.1:4437/__ursula/metrics
```

A JSON snapshot of runtime state means the node is up.
